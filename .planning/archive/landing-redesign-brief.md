# 랜딩페이지 리디자인 브리프

## 배경
- GA4 28일 데이터: 274명 랜딩 방문 → 122명 /app 진입 (55% 이탈)
- /app 진입 후에도 43%가 컨디션체크 안 함
- 게스트 첫 운동 후 흰 화면 크래시 버그는 수정 완료 (HomeScreen hooks 순서)

## 현재 랜딩 문제점
1. 영상 파일 너무 큼 (hero.mp4 27MB, is-it-right2.mp4 41MB) → 모바일 로딩 실패
2. 12스크롤 → 결정 피로
3. CTA 5개 분산 ("시작하기", "3초 만에 운동 받기", "무료로 시작" 등)
4. 스토리/증거 없이 기능 나열
5. 가격 섹션 중복 (2곳)

## 레퍼런스: Whoop (whoop.com)
- Sticky "Join Now" CTA — 스크롤해도 항상 상단 고정
- 숫자로 후킹 — "주 91분 더 운동", "2.3시간 더 수면"
- 미니멀 — 8-10섹션, 깔끔한 다크/화이트 톤
- 데이터 오버레이 — 앱 화면을 실제 사진 위에 표시
- "Backed by PhDs, worn by MVPs" — 신뢰 한 줄
- 제품 소개 구간에서만 CTA 살짝 사라졌다 다시 나타남 (킥)

## 확정된 새 구조 (4섹션)

```
[Sticky CTA: "오늘 운동 받기" — 항상 상단 고정]

Section 1: 후킹 + 숫자
  "운동 뭐 해야 할지 모르겠으면"
  + 핵심 숫자 3개 (주 3.2회 / 94% 완주 / +28% 운동량)

Section 2: 제품 데모
  앱 화면 (플랜 생성 → 운동 세션 → 리포트)

Section 3: 신뢰
  "Backed by 한체대 · ACSM · NASM" 한 줄

Section 4: 제안
  무료 시작 / 프리미엄 가격
  [오늘 운동 받기] 큰 CTA
```

## Gen Z (00-05년생) 피드백 키워드
- 3초 안에 보여줘야 함 (텍스트 읽기 싫음)
- "왜 만들었나" 스토리보다 "쓰면 이렇게 된다" 결과
- 실제 앱 화면이 제품 → 크게 보여줘야 함
- "무료로 시작" 같은 뻔한 CTA 싫음
- 다크 모드, 깔끔, 미니멀
- Whoop의 "숫자 → 증거 → 제품 → 제안" 구조가 최적

## 구현 방식
- `/landing` 경로에 새 버전 실험 (기존 `/` 유지)
- 완성 후 A/B 테스트 또는 교체
- 기존 랜딩의 FAQ, PWA 가이드, 비교 테이블 → /app 안으로 이동 또는 삭제

## 기술 체크리스트
- [ ] `/landing` 라우트 생성
- [ ] Sticky CTA (position: sticky, 스크롤 시 고정)
- [ ] 제품 소개 구간에서 CTA 일시 숨김 → 다시 등장 (Whoop 패턴)
- [ ] 영상 최적화 (41MB → 2-5MB, WebP/VP9 또는 정적 이미지)
- [ ] 4섹션 미니멀 구조
- [ ] 숫자 애니메이션 (카운트업)
- [ ] 앱 화면 데모 (스크린샷 또는 경량 영상)
- [ ] 모바일 퍼스트 (대부분 모바일 유입)
- [ ] 다크 톤 배경 (Whoop 스타일)
