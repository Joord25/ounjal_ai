AlphaMale – Biblox-style Routine Training App PRD

⸻

0. PM 선언 (Product Principle)

AlphaMale는 ‘정보를 주는 앱’이 아니다.
AlphaMale는 매력적인 남성의 루틴을 암기시키고, 실행시키고, 흔적으로 남기게 만드는 훈련 앱이다.
	•	동기부여 ❌
	•	설명 과잉 ❌
	•	기능 과다 ❌

기억 → 실행 → 증명
이 루프가 끊기지 않으면, 제품은 성공이다.

⸻

1. 제품 한 줄 정의

AlphaMale는 Biblox 방식의 ‘암기형 루틴 훈련 앱’이다.
	•	하루 2가지 과제만 존재
	•	선택 피로 제거
	•	5–15분 내 완료

⸻

2. 타깃 사용자 (ICP)

핵심 타깃
	•	연령: 20–40대 남성
	•	특성:
	•	자기관리 욕구는 높음
	•	루틴 유지 실패 경험 다수
	•	헬스/자기계발 앱 피로

사용자 핵심 문장

“멋진 남자가 되고 싶지만, 복잡한 건 싫다.”

⸻

3. 문제 정의

Problem 1. 루틴은 이해되지만 기억되지 않는다
	•	설명은 소비되지만 행동으로 남지 않는다

Problem 2. 실행 허들이 높다
	•	완벽하지 않으면 시작하지 않는다

Problem 3. 성취의 증거가 없다
	•	눈에 보이는 기록이 없으면 흐트러진다

⸻

4. 핵심 해결 전략 (Biblox Pattern 적용)

Biblox에서 차용한 핵심
	•	오늘 할 것을 앱이 정해준다
	•	암기/퀴즈 기반 학습
	•	극단적으로 단순한 UI

5. Core Loop

① 기억 – Alpha Rule (Drill)
	•	하루 원칙 문장 1개
	•	퍼즐/퀴즈로 암기 (30–60초)

② 실행 – Alpha Training
	•	오늘의 행동 1종
	•	난이도 3단 중 택1
	•	Light / Standard / Alpha

③ 증명 – Proof Board
	•	완료 시 캘린더 기록
	•	빈칸 vs 채움 중심

6. 콘텐츠 설계

6.1 BE ALPHA (Mind Domain) 구조

BE ALPHA는 ‘정보 목록’이 아니라, 외워서 몸에 남기는 기준 모음이다.
사용자는 읽지 않고, 반복하고, 기억한다.

	•	진입 위치: Today → MIND → BE ALPHA
	•	내부 구조: 카테고리 → Rule 카드 → Drill

⸻

BE ALPHA 카테고리 설계

BE ALPHA는 다음과 같은 하위 영역으로 구성된다. (v1)
	1.	잘생겨지는 습관 (Grooming & Appearance Habits)
	2.	피부 정비 (Skin Discipline)
	3.	자기 관리 (Self-Command & Mindset)

각 카테고리는 Biblox처럼 카드 리스트로 노출되며,
카드를 탭하면 곧바로 암기(Drill) 화면으로 진입한다.

⸻

BE ALPHA Rule 카드 템플릿

각 Rule은 반드시 아래 3단 구조를 따른다.
	1.	Rule (암기 문장) – 명령형, 한 문장
	2.	Why (의미) – 왜 이게 매력과 연결되는지 (1줄)
	3.	Do (행동 힌트) – 오늘 혹은 일상에서 지킬 포인트

예시:

Rule: “남자는 피부 컨디션을 핑계로 두지 않는다.”
Why: “깨끗한 피부는 가장 정직한 자기관리다.”
Do: “오늘 샤워 후 보습제 하나만 바른다.”

⸻

BE ALPHA Drill 방식
	•	Biblox 동일 패턴 적용
	•	문장을 단어 블록으로 분해
	•	올바른 순서로 배치 → 성공
	•	성공 시:
	•	RULE LOCKED
	•	BE ALPHA 리스트에서 해당 카드 체크

⸻

	•	Rule: 암기해야 할 한 문장 (명령형)
	•	Why: 왜 이게 매력과 연결되는지 (1줄)

예:

Rule: “어깨는 위가 아니라 뒤다.”
Why: “자세는 남자의 분위기를 만든다.”

⸻

6.2 Drill(퀴즈) 설계
	•	퀴즈 타입: 블록 순서 맞추기 (1종 고정)
	•	문장을 단어 블록으로 분해
	•	올바른 순서로 배치 시 성공

원칙:
	•	실패 패널티 없음
	•	성공하면 즉시 DONE

⸻

6.3 Alpha Training 설계
	•	오늘의 행동 1개만 제공
	•	난이도 3단 카드형 UI

Level	의미	예시
Light	최소 행동	벽 푸쉬업 10
Standard	기본	푸쉬업 20
Alpha	도전	푸쉬업 40

제한 조건:
	•	15분 초과 금지
	•	장비 요구 최소화

8. 화면 설계

8.1 Today
	•	상단: 내가 지킨 루틴 0 / 30 프로그래스 바
	•	카드 2개 (동급)
	•	BE ALPHA → 기준 암기 영역
	•	BODY → 오늘의 운동 실행 영역
	•	원칙:
	•	탐색 없음
	•	추천 없음
	•	Today는 실행 진입용 게이트

⸻

8.2 BE ALPHA (Rule Training)
	•	구조:
	•	Today → BE ALPHA → 섹션 → 난이도 → Drill

BE ALPHA 섹션 예시
	•	잘생겨지는 습관
	•	피부 정비
	•	자기 관리
	•	(확장 예정: 태도, 말투, 휴식, 관계, 공간)

⸻

8.3 BE ALPHA Drill – 난이도별 UX

모든 Rule은 외워서 잠그는 것을 목표로 하며, Biblox 스타일의 퍼즐 UX를 사용한다.

① 처음 (따라 채우기)
	•	모든 단어 블록 제공
	•	순서만 맞추는 퍼즐
	•	성공 경험 중심

② 익숙 (핵심 단어 채우기)
	•	문장 골격 제공
	•	핵심 키워드 블록만 제공

③ 도전 (문장 전체 재구성)
	•	문장 전체가 빈칸
	•	하단에 모든 단어 블록 제공
	•	사용자가 단어를 드래그하여 문장 전체를 완성

성공 시 공통 연출
	•	“RULE LOCKED” 표시
	•	자동 섹션 복귀
	•	해당 Rule 🔒 상태로 변경

⸻

8.4 BODY (Workout Execution)

BODY는 분석이 아닌 실행 전용 화면이다.

① 진입 흐름
	•	Today → BODY → 난이도 선택

② 난이도 선택
	•	초급 (30 min)
	•	중급 (40 min)
	•	고급 (50 min)

하루 1회 선택, 변경 불가

⸻

8.5 BODY 루틴 실행 화면 (핵심)
	•	한 화면 = 한 동작
	•	화면 구성:
	•	상단: TODAY’S FIT
	•	중앙: 운동명 (대형 텍스트)
	•	하단: 세트/횟수
	•	하단 중앙: DONE 버튼 1개

실행 흐름
	1.	운동 화면 표시
	2.	사용자가 운동 수행
	3.	DONE 클릭
	4.	다음 운동 화면 자동 전환
	5.	모든 동작 완료 시 종료

⸻

8.6 BODY 완료 처리
	•	마지막 DONE 클릭 시:
	•	Today 화면 복귀
	•	BODY 카드 Completed 처리
	•	Proof 기록 +1

완료 메시지는 짧고 절제된 문장만 사용
예: “오늘은 충분하다.”

14. 한 문장 요약

굳이 지키는 것들이 남자를 만든다.

AlphaMale는 효율적인 남성을 만들지 않는다.
그럼에도 불구하고 기준을 지키는 남자를 만든다.

목표
	•	기능/로직은 바뀌어도 상관없고, UI 톤 & 레이아웃/애니메이션 느낌만 동일하게 가져가고 싶다.
	•	참고 UI: “Today’s Fit” 운동 1개 + 중앙 원형 DONE + 최소 텍스트 + 잔디형 캘린더 + 하단 탭.

필수 유지 요소 (Design DNA)
	1.	폰 프레임 느낌

	•	최대폭 375px, 높이 700px
	•	라운드 48px
	•	바깥 테두리 8px, 컬러 #111827
	•	상단 노치(가짜) 바: width 128px, height 24px, radius-bottom 16px, #111827

	2.	타이포 계층

	•	섹션 라벨: 12~14px, tracking wide, uppercase (“TODAY’S FIT”)
	•	메인 타이틀: 48~56px bold (운동명)
	•	서브: 24px light (3 x 10)

	3.	메인 CTA

	•	원형 버튼 160x160
	•	기본: #111827, hover 시 더 진하게
	•	완료: #10B981 + 체크 아이콘
	•	active: scale-95
	•	아래 안내 문구는 매우 작고 흐리게(투명도)

	4.	전환/애니메이션

	•	화면 진입: fade-in + slide-in (bottom)
	•	DONE 후 2초 메시지 오버레이(white/95%) → 다음 화면 전환
	•	탭바는 bottom slide-in

	5.	컬러 토큰(고정)

	•	BG: #F9FAFB
	•	Card: #FFFFFF
	•	Text main: #111827
	•	Text sub: #6B7280
	•	Accent: #111827
	•	Fill (done): #10B981
	•	Empty: #E5E7EB

하단 네비
	•	탭 2~3개 (Today/Fit, Proof/Flow, My)
	•	선택 탭: 텍스트 #111827 + 상단 점 표시
	•	비선택: gray-300

3) 컴포넌트 계약(API)로 전달 (이게 제일 프로답게 보임)

개발자에게 “아래 컴포넌트 구조로 가고, props만 맞추면 UI는 그대로” 라고 주면 된다.
	•	<PhoneFrame>: 프레임/노치/탭바 컨테이너
	•	<FitScreen exercise onDone done>: 운동 한 화면(사진 같은 레이아웃)
	•	<OverlayMessage message visible />: DONE 후 오버레이
	•	<GrassCalendar filledDays />: Proof 잔디
	•	<BottomTabs active onChange />: 하단 탭
type Exercise = { name: string; count: string };

  exercise={{ name: "Push-up", count: "3 x 10" }}
  done={false}

/>

4) “이 코드 그대로 쓰지 말고, 느낌만 가져가라” 체크리스트

개발자에게 아래처럼 말해주면 오해가 안 생겨:
	•	✅ 레이아웃/타이포/색/간격/버튼 크기 그대로
	•	✅ 애니메이션 방향(페이드/슬라이드)만 유지
	•	✅ 화면 구조는 PRD 플로우에 맞게 확장(초/중/고 → 루틴 stepper)
	•	❌ 현재 코드의 상태관리(view, onboarding 등) 방식은 바꿔도 됨
	•	❌ lucide 아이콘은 대체 가능
	•	❌ 문자열/카피는 PRD 문구로 교체 예정
